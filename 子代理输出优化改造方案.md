# 子代理输出优化改造方案

## 📋 背景

当前 mini_claude 的子代理系统存在以下问题：

1. **上下文污染**：子代理的所有文本输出（包括中间过程、废话）都会被捕获并返回给主代理
2. **缺乏过滤机制**：无法区分"最终结果"和"中间过程"
3. **与生产级框架差距**：OpenAI Agents SDK 支持 `custom_output_extractor`，而 mini_claude 没有类似机制

### 参考：生产级框架的做法

| 框架 | 返回内容 | 提取方式 |
|------|---------|---------|
| **OpenAI Agents SDK** | `result.final_output` | `custom_output_extractor` 自定义提取器 |
| **Claude Code** | 子代理的文本输出 | System Prompt 约束输出格式 |
| **mini_claude（当前）** | 所有文本输出的拼接 | 无过滤 |

---

## 🎯 改造目标

1. **添加结构化输出标识符** — 子代理使用 `<result>` 标签标记最终输出
2. **添加输出提取器** — 从原始输出中提取标识符内的内容
3. **修改 System Prompt** — 约束子代理的输出格式
4. **保持向后兼容** — 如果子代理不遵循格式，回退到原始文本

---

### 步骤 1：修改 `subagent.py` — 添加输出格式指导

**文件**：`example/mini_claude/subagent.py`

**修改内容**：

1. 添加通用的输出格式常量
2. 在 `get_sub_agent_config()` 中统一注入输出格式指导（确保自定义代理也能获得）

```python
# subagent.py

# ─── 输出格式指导 ────────────────────────────────────────────

OUTPUT_FORMAT_INSTRUCTION = """

=== OUTPUT FORMAT ===
After completing your task, wrap your final summary in these exact tags:

<result>
Your concise findings here. Include:
- Key discoveries or actions taken
- File paths with line references (if applicable)
- Essential information for the caller
</result>

IMPORTANT:
- Everything outside <result> tags will be DISCARDED
- Focus on tool usage during execution, minimize commentary
- Only the content inside <result> tags reaches the main agent
- Use ONE <result> tag only. If reporting multiple items, include all inside a single tag.
"""

# ─── Main config function ───────────────────────────────────


def get_sub_agent_config(agent_type: str) -> dict:
    """Return {system_prompt, tools} for the given agent type."""
    custom = _discover_custom_agents().get(agent_type)
    if custom:
        if custom["allowed_tools"]:
            tools = [t for t in tool_definitions if t["name"] in custom["allowed_tools"]]
        else:
            tools = [t for t in tool_definitions if t["name"] != "agent"]
        # 统一注入输出格式指导（包括自定义代理）
        system_prompt = custom["system_prompt"] + OUTPUT_FORMAT_INSTRUCTION
        return {"system_prompt": system_prompt, "tools": tools}

    read_only = [t for t in tool_definitions if t["name"] in READ_ONLY_TOOLS]

    if agent_type == "explore":
        return {"system_prompt": EXPLORE_PROMPT + OUTPUT_FORMAT_INSTRUCTION, "tools": read_only}
    elif agent_type == "plan":
        return {"system_prompt": PLAN_PROMPT + OUTPUT_FORMAT_INSTRUCTION, "tools": read_only}
    else:  # general
        return {"system_prompt": GENERAL_PROMPT + OUTPUT_FORMAT_INSTRUCTION, "tools": [t for t in tool_definitions if t["name"] != "agent"]}
```

> **注意**：内置 Prompt 常量（EXPLORE_PROMPT、PLAN_PROMPT、GENERAL_PROMPT）保持原样不变，输出格式指导通过 `get_sub_agent_config()` 统一注入。

### 步骤 2：修改 `agent.py` — 添加输出提取逻辑

**文件**：`example/mini_claude/agent.py`

**修改内容**：

1. 添加 `import re`
2. 添加 `_extract_result` 静态方法
3. 修改 `run_once` 方法，添加提取逻辑

```python
# agent.py

import re

class Agent:
    # ... 现有代码 ...
    
    # ─── Output extraction ──────────────────────────────────
    
    @staticmethod
    def _extract_result(raw_text: str) -> str:
        """从原始输出中提取 <result> 标签内的内容。
        
        如果找到 <result> 标签，返回最后一个标签内的内容（假设是最终版本）；
        否则返回原始文本（兼容不遵循格式的子代理）。
        
        Args:
            raw_text: 子代理的原始输出文本
            
        Returns:
            提取后的内容，或原始文本（如果未找到标签）
        """
        matches = re.findall(r'<result>\s*(.*?)\s*</result>', raw_text, re.DOTALL)
        if matches:
            # 取最后一个，假设是子代理的最终修正版本
            return matches[-1].strip()
        # 兼容模式：如果没有标识符，返回原始文本
        return raw_text
    
    # ─── Sub-agent entry point ──────────────────────────────
    
    async def run_once(self, prompt: str, keep_raw: bool = False) -> dict:
        """子代理一次性运行接口。
        
        Args:
            prompt: 任务描述
            keep_raw: 是否保留原始输出（调试用）
            
        Returns:
            {
                "text": str,           # 提取后的结果
                "raw_text": str | None, # 原始输出（仅当 keep_raw=True）
                "tokens": {
                    "input": int,
                    "output": int,
                }
            }
        """
        self.state.output_buffer = []
        prev_in = self.state.total_input_tokens
        prev_out = self.state.total_output_tokens
        
        await self.chat(prompt)
        
        raw_text = "".join(self.state.output_buffer)
        self.state.output_buffer = None
        
        # 提取 <result> 标签内的内容
        result_text = self._extract_result(raw_text)
        
        result = {
            "text": result_text,
            "tokens": {
                "input": self.state.total_input_tokens - prev_in,
                "output": self.state.total_output_tokens - prev_out,
            },
        }
        
        # 可选保留原始输出
        if keep_raw:
            result["raw_text"] = raw_text
        
        return result
```

```markdown
### 步骤 4：流式输出捕获与一次性运行接口设计

#### 4.1 基础实现：输出拦截与缓冲区

... (现有内容：_emit_text 和 run_once 基础实现) ...

#### 4.2 进阶优化：结构化输出与提取

##### 为什么做

当前实现会捕获子代理的**所有**文本输出，包括中间过程的分析、工具调用说明等。
这会导致主代理的上下文被大量中间信息污染。

参考 OpenAI Agents SDK 的设计，我们可以通过结构化输出来解决这个问题：
- 子代理使用 `<result>` 标签标记最终输出
- 主代理只提取标签内的内容，过滤掉中间过程

##### 做什么

1. **修改 System Prompt**：在 `subagent.py` 中添加输出格式指导

\`\`\`python
OUTPUT_FORMAT_INSTRUCTION = """

=== OUTPUT FORMAT ===
After completing your task, wrap your final summary in these exact tags:

<result>
Your concise findings here.
</result>

Everything outside <result> tags will be discarded.
"""
\`\`\`

2. **添加提取逻辑**：在 `agent.py` 中实现 `_extract_result` 方法

\`\`\`python
@staticmethod
def _extract_result(raw_text: str) -> str:
    matches = re.findall(r'<result>\s*(.*?)\s*</result>', raw_text, re.DOTALL)
    if matches:
        return matches[-1].strip()  # 取最后一个，假设是最终版本
    return raw_text  # 兼容模式
\`\`\`

3. **修改 `run_once`**：调用提取逻辑

\`\`\`python
async def run_once(self, prompt: str, keep_raw: bool = False) -> dict:
    # ... 运行子代理 ...
    raw_text = "".join(self.state.output_buffer)
    result_text = self._extract_result(raw_text)
    
    result = {"text": result_text, ...}
    if keep_raw:
        result["raw_text"] = raw_text
    return result
\`\`\`

##### 注意什么

- **向后兼容**：如果子代理不遵循格式，`_extract_result` 会返回原始文本
- **调试支持**：`keep_raw=True` 可以保留原始输出，方便调试
- **System Prompt 统一注入**：通过 `get_sub_agent_config()` 统一添加，确保自定义代理也能获得格式指导
- **多标签处理**：如果子代理输出多个 `<result>` 标签，取最后一个（假设是最终修正版本）

##### 设计权衡

| 方案 | 优点 | 缺点 |
|------|------|------|
| 结构化输出（本方案） | 精确过滤，只返回结果 | 需要子代理遵循格式 |
| 原始输出（当前） | 简单，无需格式约束 | 带回所有中间过程 |
| 自定义提取器 | 灵活，可适配任意格式 | 实现复杂，教学成本高 |

##### 多标签处理策略

| 策略 | 做法 | 适用场景 |
|------|------|----------|
| 取第一个 | `re.search` 只匹配第一个 | 子代理可能"废话"在前，真正结果在后 |
| **取最后一个（采用）** | `re.findall` 取最后一个 | 子代理可能修正结果，最终版本最准确 |
| 全部拼接 | `re.findall` 拼接所有 | 多文件/多任务场景，每个都是独立结果 |

**选择理由**：子代理输出多个标签时，通常意味着修正或补充，最终版本最可靠。
```

## ⚠️ 注意事项

1. **System Prompt 统一注入**：在 `get_sub_agent_config()` 中统一添加 `OUTPUT_FORMAT_INSTRUCTION`，确保内置代理和自定义代理（`.claude/agents/*.md`）都能获得格式指导
2. **多标签处理策略**：使用 `re.findall` + 取最后一个，假设子代理可能修正结果，最终版本最准确
3. **测试场景**：
   - 子代理遵循格式 → 正确提取
   - 子代理不遵循格式 → 回退到原始文本
   - 子代理输出多个 `<result>` 标签 → 取最后一个
   - `keep_raw=True` → 返回原始文本字段
4. **文档更新**：需要同步更新教程文档，说明新的输出格式要求
